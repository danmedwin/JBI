📋 Next Step: Page Numbering
The Word document above uses the source siddur's page numbers as markers. When the output is longer than the original (e.g., large font), a single source page may span multiple Word pages. The Page Numberer script detects these overflows and adds lettered suffixes (e.g., 42a, 42b) so every Word page has a unique number.
Option A: Upload the downloaded .docx to claude.ai and ask Claude to run the page numberer on it.
Option B: Run the script locally on your computer (requires Python 3 and LibreOffice):
Requirements: Python 3, LibreOffice, and: pip3 install python-docx pypdf lxml
Usage: python3 siddur_page_numberer.py my_siddur_processed.docx
Output: Creates a _numbered.docx file with lettered page suffixes where needed.
How it works
1. Upload a siddur PDF — pages are rendered as high-resolution images.
2. AI auto-detects the siddur layout (you can override).
3. Adjust font size, add notes, then hit Begin Processing.
4. Each page is analyzed: Hebrew with nikkud, transliteration, translation, and more.
5. A Word document is generated with named paragraph styles and collected footnotes.
