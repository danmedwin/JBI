Siddur OCR Project

AI-powered tools for converting printed prayer books into accessible, large-print Word documents — preserving Hebrew with nikkud, transliteration, translation, and page structure.

Tools
1
Siddur OCR Processor
Upload a siddur PDF and let Claude's vision AI extract Hebrew liturgy, transliteration, translation, instructions, and footnotes into a formatted Word document. Requires an Anthropic API key.
Web App→
2
Page Numberer Script
Python post-processor that adds accurate lettered page numbers (e.g., "42a", "42b") to the Word output. Uses LibreOffice to detect actual page breaks. Run it via Claude chat — just upload the .docx and the script.

Workflow
1

Process PDF with the OCR app

Open Siddur OCR Processor, enter your API key, upload a siddur PDF, and click Begin Processing. Download the generated .docx file.

2

Add page numbers via Claude

Open claude.ai, upload the .docx from step 1 and siddur_page_numberer.py, then ask: "Run the page numberer on this document." Claude will execute the script and return the numbered file.

3

Review and adjust

Open the numbered document in Word or Google Docs. Check page number placement — occasionally a break may land a sentence too early or late. You can ask Claude to adjust specific page numbers if needed.

Supported Layouts

Mishkan T'filah

CCAR Press / Reform movement. Parallel columns: Hebrew right, transliteration left. ALL CAPS section headers, dual page numbers (e.g., "2 [120]").

Kol Haneshamah

Reconstructionist Press. Block Hebrew, running headers, author attributions, responsive readings, divine names in small caps, bottom section labels.

②
